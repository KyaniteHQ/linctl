# Collection-key projection stays type-derived with an allowlist fallback

linctl's `--fields` projection over a list page derives the collection field from the page type by reflection: `collectionKeyForPage` finds the single exported slice field and returns its JSON name, and every list scaffold annotates its command with that key for the projector to read. A hand-maintained `collectionKeys` allowlist remains only as the fallback for pages with more than one slice field, where the reflection is deliberately ambiguous, and for detail responses that carry an incidental array which must not be projected per element. Because the common single-collection list page is handled automatically by the type, the allowlist is a deliberate guard for the ambiguous cases rather than a per-entity maintenance burden, so it is kept as-is instead of being replaced by a per-type registration.
