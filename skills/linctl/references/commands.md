# linctl command reference

Schema-aligned Linear CLI

Generated by `scripts/gen-skill.go` from the linctl command tree. Do not edit by hand; run `task gen-skill` to refresh it. This is the full command inventory linked from `SKILL.md`.

## Global flags

```
      --compact            emit compact JSON when --json is set
      --debug              emit debug diagnostics to stderr
      --fail-on-empty      exit non-zero when a list result is empty
      --fields string      comma-separated JSON fields to emit
      --format string      human output format: minimal, compact, or full (default "compact")
      --id-only            emit only Linear ids
      --json               emit JSON output
      --order string       sort order: asc or desc (default "asc")
      --org string         pinned Linear organization id
      --profile string     config profile to load
      --project string     pinned Linear project id
      --quiet              suppress successful output
      --sort string        JSON field to sort list output by
      --team string        pinned Linear team key
      --team-id string     pinned Linear team id
      --timeout duration   request timeout (default 30s)
```

### linctl issue create

Create an issue in the pinned target

```
linctl issue create [flags]
```

Flags:

```
      --assignee string           assign the issue to a user id
      --description string        issue description
      --description-file string   read issue description from file
      --dry-run                   render the assembled issue without creating it
      --due-date string           set the due date (YYYY-MM-DD)
      --estimate int              set the estimate (validated against team config)
      --label stringArray         attach a label by id (repeatable)
      --parent string             create as a sub-issue of a parent issue id
      --priority string           set priority (urgent/high/medium/low/none or 0-4)
      --section stringArray       fill a markdown section: NAME=VALUE (repeatable)
      --state string              set workflow state type (e.g. started, completed)
      --status string             alias for --state
      --template string           apply a Linear template by id for title/description defaults
      --title string              issue title
```
